# Security Policy

## Supported versions

Postyard Templates is pre-1.0. Security fixes are released against the latest
published version only.

| Version  | Supported          |
| -------- | ------------------ |
| latest   | :white_check_mark: |
| older    | :x:                |

This applies to every published library in this project, across all supported
languages.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately through GitHub's private vulnerability reporting:

➡️ **[Report a vulnerability](https://github.com/postyard/templates/security/advisories/new)**
(repository **Security** tab → **Advisories** → **Report a vulnerability**)

Please include as much of the following as you can:

- The affected library and version.
- The type of issue and the impact you believe it has.
- Steps to reproduce, or a proof of concept.
- Any known mitigations or workarounds.

## What to expect

- We aim to acknowledge your report within **1 week**.
- We will keep you informed of our progress and confirm the issue.
- We aim to release a fix and publish an advisory within **90 days** of
  confirmation, and will coordinate disclosure timing with you.

We are grateful for responsible disclosure and will credit reporters in the
published advisory unless you ask us not to.
